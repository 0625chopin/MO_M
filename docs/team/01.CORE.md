# 01. CORE

**A팀** 소속이며 주 담당은 **공통 기반·타입/데이터 경계**다. ROADMAP에서 되돌리기가 가장 비싼 것들 — 디렉터리 규약(**R-007**), 도메인 타입(**NFR-035**), 데이터 접근 레이어 경계(**NFR-034**·037), 스키마와 RLS(**NFR-011**·032) — 이 한 계통으로 이어져 있어 **경계를 세운 사람이 실데이터로 교체하는 것**이 인계 비용을 가장 줄인다.

## 배정 근거

- **경계를 정의한 사람이 경계를 교체한다.** Mock 데이터 접근 레이어(**NFR-034**·037)를 설계한 사람이 그것을 Supabase 조회로 바꾸는 Task를 이어 받으면, **D-030**이 요구한 "UI 컴포넌트를 수정하지 않고 조회부만 교체"가 실제로 성립하는지 스스로 검증하게 된다. 여기서 UI를 고쳐야 한다면 자기가 세운 경계가 샌 것이다
- **스키마와 RLS는 되돌리기가 가장 비싸다.** **NFR-032**가 투표 기록의 소급 변경을 금지하므로 스키마 설계 시점이 마지막 기회이고(**D-033**), Supabase는 RLS 미설정 테이블이 사실상 전체 공개다(**NFR-011**). 이 계통을 도메인 타입을 정의한 사람에게 준 이유다
- **채팅을 보조로 겸임한다.** 구독 인터페이스(**D-030 ②**)는 데이터 경계와 같은 성격의 추상화이며, `subscribeToRoom` 형태를 쓰는 쪽과 정의하는 쪽이 같은 계통에 있는 편이 인터페이스가 덜 샌다
- **이 사람에게 몰리면 안 되는 것**: `🚧`(차단) Task를 주지 않는다. 이 사람이 지는 구간(Phase 4 전반)은 **뒤따르는 Task 전부의 선행**이라 하나만 막혀도 나머지 3명이 동시에 멈춘다. **I-016**에 막힌 인증 연결 Task는 다른 사람이 진다
- **주의**: Phase 4 전반을 혼자 연속으로 진다. **즉시 인계할 사람이 없는 구간**이므로 진행 상황을 주 단위로 공유하고, 스키마·RLS 결정은 반드시 리뷰어와 대조한다
