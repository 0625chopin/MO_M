# mo_im 개발 일정

팀원 **4명**을 **2개 팀**으로 편성해 ROADMAP의 잔여 Task **45건**(XL 분할 후 **56건**)·잔여 공수 **321.7인일**을 배정한 결과이며, 작업 종료 예정일은 **2027-06-20**, 버퍼를 포함한 최종 종료 예정일은 **2027-08-15**다. **이 일정의 길이를 결정하는 것은 인원이 아니라 크리티컬 패스 137.6인일(34.4주)이다** — 총 공수를 4명으로 나누면 20.1주지만, 직렬 의존 사슬 때문에 47주가 필요하고 그 결과 **전체 가용 인일의 약 57%가 유휴**로 남는다. **이 프로젝트는 4명 규모의 일감이 아니다**(**I-019**·**R-020**).

## 개요

- **인원**: 4명 / 2개 팀 (팀당 2명)
- **기간**: 2026-07-27 ~ 2027-08-15 (총 55주, 버퍼 8주 포함 / 작업 47주)
- **잔여 공수**: 321.7인일 / **크리티컬 패스**: 137.6인일 = **34.4주**
- **팀별 부하 편차**: 6.8% (목표 ≤10%) / **개인별 부하 편차**: 9.9% (목표 ≤15%)

### 이 문서의 위치와 규칙

- **스코프의 단일 소스는 [`docs/ROADMAP/ROADMAP.md`](../ROADMAP/ROADMAP.md)다.** Task 존재·제목·구현 사항·수락 기준·Phase 구성이 이 문서와 어긋나면 ROADMAP이 이긴다
- **전체 일정의 단일 소스는 이 문서다** — 팀 편성·개인 부하·주차별 배치·크리티컬 패스·균등 검증. 일정을 ROADMAP에 역주입하지 않는다
- **사람의 단일 소스는 [`docs/team/`](../team/)이다** — 역할·배정 근거·가용 인일·리뷰 짝
- **개인별 배정 상세의 단일 소스는 [`docs/ROADMAP/team/`](../ROADMAP/team/)이다** — 담당 Task의 계산식·주차·선행 대기. 이 문서는 Task↔담당 대응까지만 적는다
- **진행률·완료 개수 같은 갱신되는 집계값은 적지 않는다.** 상태는 각 Task 줄의 ✅ 표시 하나로만 관리한다
- ROADMAP이 바뀌면 이 문서가 낡은 것이다. ROADMAP을 고치지 말고 **일정을 재산출한다**

### 공수 산정을 불릿 기반으로 하지 않은 이유

기본 규칙(잔여 불릿 수 × 0.5인일 × Phase 계수 × Task 계수)을 그대로 적용하면 **v0.1(Task 001~025) 합계가 74.7인일**로 나온다. 이는 `PRD-validation.md` §6.3의 **168.5인일**의 44%이며, 4명 기준으로 환산하면 일정 자체가 무의미해진다.

괴리의 원인은 ROADMAP의 서술 밀도다. ROADMAP은 Task 하나를 **성격 단위**로 적고 분량 단위로 적지 않는다 — 예컨대 Task 013의 불릿 5개 안에 "원자 15종"이 들어 있고, Task 015의 불릿 5개가 화면 4개와 컴포넌트 다수를 담는다. 불릿 개수는 **작업량이 아니라 결정 항목 수**의 신호다.

그래서 다음 순서로 산정했다.

1. **`PRD-validation.md` §6.2 영역별 표의 각 줄을 Task에 매핑**한다. §6.2는 화면 수·컴포넌트 수·4상태·뷰포트 3종·WCAG 항목을 세어 만든 값이라 실물 근거가 있다
2. §6.2가 **Task 009(순수 규칙 모듈) 소관인 항목을 기능 영역에 계상**한 것을 ROADMAP 기준으로 되돌린다 — 권한 매트릭스 3.0(크루·멤버십에서), 판정·정족수 경계 케이스 3.0(투표에서), 색 배정 해시 1.5(캘린더에서) → 합 7.5인일을 **Task 009로 이관**한다. 영역 소계 합은 보존된다
3. §6.2에 줄이 없는 Task 004·005·025는 **불릿 기반으로 보완**한다(합 6.9인일). §6.2가 이 셋을 다루지 않은 것이지 비용이 0인 것이 아니다
4. Phase 4는 §6.4의 **v0.2 개략 59~89인일의 중앙값 74.0인일**을 항목별로 Task에 배분하고, §6.4가 다루지 않는 Task 026·037·038을 불릿 기반으로 보완한다(합 7.5인일)
5. **Phase 5·6은 `PRD-validation.md`에 산정이 없다.** v0.1 기능 영역 실측 단가 **FR당 3.1인일**(기능 영역 120.5인일 ÷ FR 39건)을 Phase 5·6이 다루는 FR 15건에 적용하고, 관리자 콘솔(SC-21)·성능 반복 측정처럼 FR 단가로 잡히지 않는 항목만 별도 계상한다

**교차 검증**: 이 방식의 Phase 1~4 합계는 **256.9인일**이며, `PRD-validation.md` §6.4의 결론 "v0.1 + v0.2 ≈ 255~300인일"과 일치한다. v0.1분 175.4인일은 §6.3 기본 합계 168.5인일 + 보완 6.9인일과 정확히 같다.

**이 전제 중 하나라도 바뀌면 일정을 재산출한다.**
