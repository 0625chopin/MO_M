# ISSUES

개발 중 발견한 **미결 이슈와 개선사항**을 기록한다. 이 파일이 이슈 번호의 **단일 소스**다.

- **다음 이슈 번호: I-001** (등재할 때마다 이 줄을 갱신한다)
- 확정된 **결정**은 여기가 아니라 `docs/require/06-prioritization-and-risks.md` 6.3절 결정 기록(D-\*)에 쓴다. 결정과 미결을 같은 곳에 두지 않는다.
- **제보는 전원**, **반영(이 파일 편집)은 1팀 코어·품질팀**이 한다. 일차 마감 교차 점검에서 나온 항목은 팀장이 직접 등재하기도 한다.

## 기록 형식

```md
### I-001 · 제목 한 줄

- **상태**: 열림 | 진행중 | 해결됨 | 보류 | 무효
- **영역**: 예) UI / 데이터 / 빌드 / 접근성 / 문서
- **제보**: 이름 (YYYY-MM-DD)
- **내용**: 무엇이 문제인가. 재현 경로나 해당 파일 경로를 포함한다.
- **영향**: 방치했을 때 무엇이 깨지거나 느려지는가.
- **후속**: 해결 시 커밋/PR 링크, 결정으로 승격됐다면 D-\* 번호.
```

상태를 `해결됨`·`무효`로 바꿀 때도 항목을 지우지 않는다. 같은 문제가 다시 올라왔을 때 이전 판단을 찾을 수 있어야 한다.
