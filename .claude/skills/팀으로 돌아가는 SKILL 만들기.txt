

agent team을 사용해 병렬로 업무를 진행할  팀원을 [@docs/team/ 폴더 하위 파일갯수] 명 구성할겁니다.

* 팀 *
  팀1 | @docs/team/ 하위 파일들 (형태는 01.팀명.md, 02.팀명.md)

* 업무 스케쥴 *
  팀1 | @docs/ROADMAP/team/ 하위 파일들 (형태는 01.팀명.md, 02.팀명.md)

으로 구성되며. 

팀원은 해당일차에 필요한 팀들만 필요 시점에 호출되어 수행합니다. 
N일차에 다른팀의 선행 작업이 필요할 경우 선행작업 완료 후 spawn 합니다.
spawn 시 Sonnet model을 사용 합니다.

작업진행 요청은 팀장만 진행 할것이며 "N일차 진행" 형식으로 할것입니다. 
예) 3일차 진행, 3일차 진행해주세요.

각각의 팀원은 본인이 필요한 md파일만 읽어야 합니다. (필요시에만 ROADMAP.md 파일을 참조하세요)

팀장으로 부터 요청이 들어오면 shrimp tasks manager 통해 계획을 먼저 수립 합니다.


N일차가 완료되면 N일차에 작헙한 팀들은 작업내용을 상호 공유하고 교차 점검 후 이슈 여부를 체크 합니다.
서로 교차 검증 후 이상이 없을시 팀장한테 작업내용을 전달합니다.

교차검증 및 팀장검증시 이슈가 있을경우 이슈가 사라질때까지 아래 로직을 반복합니다.
  1. 이슈 발생 팀원에게 내용전달
  2. 이슈를 전달 받은 팀원은 수정 작업 진행
  3. 수정 완료 후 연관된 팀원 교차 재 검증 
  4. 이슈 해결시 팀장에게 내용전달
  5. 팀장이 전체적인 테스트 진행
  6. 이슈 발생시 1번으로 회귀, 이슈 없을시 검증작업 종료


각각의 일차별 모든 업무가 이슈 없이 끝날경우 아래를 진행해주세요
- 팀원들 업무 스케쥴 파일을 update 한다.
- 필요시에만 담당업무 파일을 update 한다.
- 팀원들을 모두 종료 해주세요.
- docs/dailyWorkLog에 N_Day.md 파일로 정리 후 남겨줘
- remote git repository 에 커밋/푸쉬를 진행 해주세요.


사용법 : N일차 실행해 주세요. N일차 진행해 주세요.
